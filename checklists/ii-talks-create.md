# II Talks — Event Creation Checklist

Use this checklist before the `ii-talks` CREATE mode runs any WooCommerce or Mailchimp actions. All items must be confirmed before Rosy will proceed with live uploads.

---

## Talk & Event Details
- [ ] Talk title is confirmed and matches the series naming convention
- [ ] Venue is confirmed (UUFVB / SOD / Online)
- [ ] Event date is confirmed (MM/DD/YYYY)
- [ ] Ticket price is confirmed
- [ ] Venue capacity is appropriate for expected attendance

## Canva Assets
- [ ] **Ticket image** is ready — will be used as the featured image on the WooCommerce ticket product
- [ ] **Event image** is ready — will be used on the /events page
- [ ] **Recording image** is ready (or confirmed to be reused from a prior venue for this talk title)
- [ ] Images are uploaded to WordPress Media Library or placed in `inbox/` and ready to attach

## Ticket Product Draft
- [ ] Product name follows the convention: `TICKET – [Talk Title] at [Venue Abbreviation] ([M/D])`
- [ ] Short description is complete and proofread
- [ ] Full description is complete and proofread
- [ ] No placeholder text remaining in the draft
- [ ] Tax status is set to "None" (event tickets are not taxable services in FL)
- [ ] Product visibility is set to "Public"
- [ ] Featured product flag is confirmed — product should appear on `/events`

## Pre-Show Email Draft
- [ ] Email subject line is written and does not sound spammy
- [ ] Preview text is complete
- [ ] Body copy is written and proofread
- [ ] Correct ticket URL is included in the CTA
- [ ] Audience segment for this email is confirmed with Vanessa
- [ ] Proposed send date is at least 5 days before the event

---

*Clear all items above, then type GREEN LIGHT to proceed with WooCommerce product creation and Mailchimp draft.*
